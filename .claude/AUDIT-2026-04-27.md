## Findings тАФ triaged

### ЁЯФ┤ Broken тАФ blocks user

**1. Service-role key in `.env.local` was invalid тЖТ every recording upload returned 500.** тЬЕ FIXED (user rotated key)

- **Where it surfaced:** click Start Recording in `interview-canvas.tsx`, speak, click Stop. `POST /api/interviews/upload` returned 500. Canvas showed `Upload failed (500)`.
- **Server log:** `[upload] interviews update failed: { message: 'Invalid API key', hint: 'Double check your Supabase anon or service_role API key.' }` at `apps/api/src/app/api/interviews/upload/route.ts:84`.
- **Independent confirmation:** direct curl to `${SUPABASE_URL}/auth/v1/admin/users` with the same key returned **401**, while the anon key returned **200** on `/auth/v1/health`.
- **Cause:** service-role JWT was rotated in Supabase dashboard *after* it was last set in Vercel. `vercel env pull` got the stale value.
- **Resolution:** user rotated the key, re-pulled Vercel env, replaced in local `.env.local`. Re-tested in this audit; auth now passes through. Vercel project env still needs verification (action item below).

**1b. Supabase Storage rejects `audio/webm;codecs=opus` (415) тЖТ upload fails after auth passes.** тЬЕ FIXED (code)

- **Where it surfaced:** after the service-role fix, retried recording тЖТ got `[upload] storage upload failed: mime type audio/webm;codecs=opus is not supported (statusCode: 415)`. Server returned 500. This was the *second half* of the production bug chain тАФ even with valid auth, every Chromium recording would have 415'd.
- **Root cause:** Chromium's `MediaRecorder` produces chunks labeled `audio/webm;codecs=opus`. The recording bucket's MIME allowlist matches the bare `audio/webm`, so the codec specifier fails the check.
- **Fix landed in this audit:** `apps/api/src/app/api/interviews/upload/route.ts:138` тАФ strip `;codecs=...` before passing `contentType` to `serviceSupabase.storage.upload`. Added regression test in `apps/api/src/__tests__/unit/interviews-upload-route.test.ts` that asserts `audio/webm;codecs=opus` arrives at Storage as `audio/webm`.
- **Verified end-to-end:** recording flow now returns 201, status flips to `completed`, `recording_path` is set, transcript array is written. Confirmed by querying Supabase directly after a Playwright-driven Start тЖТ Stop cycle.

**1c. Storage-failure handler doesn't reset `status='live'` тЖТ UI strands on "End conversation".** тЬЕ FIXED

- After a storage upload failure, the route only set `upload_status='failed'` but left `status='live'`. The canvas UI then showed "End conversation" instead of "Start recording", with no way for the user to retry.
- Fixed alongside the MIME bug at `apps/api/src/app/api/interviews/upload/route.ts:147` тАФ failure handler now also sets `status='scheduled'`, mirroring `markFailed`.

**2. Stripe checkout 500 тЖТ empty body тЖТ client crash.** тЬЕ FIXED

- **Where it surfaced:** click Upgrade to Pro on `/billing`. `POST /api/stripe/checkout` returned 500 with empty body. Browser threw `SyntaxError: Failed to execute 'json' on 'Response': Unexpected end of JSON input`.
- **Three layered bugs in one click:**
  1. `NEXT_PUBLIC_APP_URL` was unset тЖТ `success_url: "undefined/billing?success=true"` тЖТ Stripe rejected with `code: 'url_invalid', param: 'success_url'`.
  2. `apps/api/src/app/api/stripe/checkout/route.ts` had no try/catch around `stripe.checkout.sessions.create` тЖТ unhandled exception тЖТ Next returns 500 with empty body.
  3. `apps/desktop/src/app/(app)/billing/page.tsx` `handleUpgrade` did `await res.json()` without checking `res.ok` тЖТ crash on empty body.
- **Fix landed in this audit:**
  - `route.ts` (checkout + portal): try/catch with structured `{ error, stage, detail, code, param }` response. `appUrl` falls back to request origin when env is unset.
  - `billing/page.tsx`: added `readErrorDetail` helper, `handleUpgrade` and `handleManageBilling` now check `res.ok` and `window.alert` the structured detail.
  - `.env.local`: added `NEXT_PUBLIC_APP_URL=http://localhost:3000` (also needs to be set in Vercel for prod).
- **Verified:** retried Upgrade to Pro тЖТ opened a real Stripe test-mode checkout session in a new tab.

**3. `OPEANI_API_KEY` typo in `.env.local`.** тЬЕ FIXED

- The OpenAI client looks for `OPENAI_API_KEY`. The typo means every analyst/agent call falls through to "no key" and silently no-ops.
- **Fix:** renamed in-place via `sed`. Also needs renaming in Vercel env if the typo lives there too.

## Findings added in this audit pass

**9. Copilot endpoint had no plan gate.** тЬЕ FIXED

- `apps/api/src/app/api/projects/[id]/interviews/[interviewId]/copilot/route.ts` тАФ `liveAICopilot: true` exists in `plans.ts` for Pro/Max but was never checked server-side. Free users were able to drain LLM credits via the SSE stream.
- Fix added a `subscriptions.plan` lookup; free returns `402 { code: 'PLAN_LIMIT', feature: 'liveAICopilot' }`. Locked in by E3 specs.

**10. `saveAnalyst` had the same silent-drop pattern as the recording route.** тЬЕ FIXED

- `apps/api/src/lib/agents/analyst/nodes.ts:241` did `.update().eq()` against `analyst_documents`. If no row pre-existed for the project (e.g. trigger missed, table cleaned), the update affected 0 rows and the route logged a generic error then continued тАФ pipeline reported success but no analyst document landed.
- Fix: switched to `.upsert({ ...payload }, { onConflict: 'project_id' }).select("project_id")`. Now the row lands whether or not a stub existed, and 0-row upserts surface as an explicit error log. Added `customer_language` to the payload (required by `AnalystDocument` type).

**11. Free-tier limits are not enforced anywhere.** ЁЯФ┤ NOT YET FIXED тАФ needs design (see complicated-fixes summary)

- `apps/desktop/src/lib/billing/plans.ts` defines `projects: 1`, `interviewsPerProject: 2`, `analystRuns: 1` for free, but `withinLimit()` is only referenced in unit tests. Direct Supabase inserts from `useCreateProject` / `useCreateInterview` bypass any check. The analyst route also doesn't count prior runs.
- Locked in via three regression-lock specs (B4, C5) so when enforcement ships these specs flip to assert the gate.

**12. Project & interview edit/delete have no UI surface.**

- `useUpdateProject` is wired (settings page); `useDeleteProject` is exported but never called from a component (`grep` returns zero non-test usages).
- `useUpdateInterview` and `useDeleteInterview` exist but neither is called from any component тАФ no edit/delete UI on interview rows.
- Documented; not a CI-coverable bug since the hooks themselves work, just inaccessible.

**13. Google OAuth `state` param contains the user's Supabase bearer token.**

- `apps/api/src/app/api/calendar/google/connect/route.ts:28` base64-encodes `{ user_id, token }` into the `state` parameter. That URL transits through Google, browser history, and any logging proxy on the redirect тАФ the token leaks.
- Fix is non-trivial: needs a server-side nonce table (or short-TTL Redis/Supabase cache) to associate the OAuth state with a user. Documented in complicated-fixes summary.

**14. `apps/desktop/src/components/upgrade-button.tsx` is dead code.** (also flagged in earlier pass)

- Component still present, no callers, calls relative `/api/stripe/checkout` (would hit desktop, not api app), no Bearer token. Safe to delete.
